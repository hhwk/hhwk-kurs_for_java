# Отчётная машина

Консольное Java-приложение для автоматического заполнения Excel-отчётов данными из `.md` файлов с таймкодами. Умножает таймкоды на коэффициент скорости воспроизведения и записывает результат в нужный лист книги.

---

## Требования

| Компонент | Версия |
|-----------|--------|
| JDK | 11+ (на машине установлен Eclipse Adoptium JDK 25) |
| Maven | 3.6+ (только для сборки) |

---

## Быстрый старт

```cmd
run.bat FIRE_FORCE_S2_EP2 1.5
run.bat Крупняк 1.5
```

Первый аргумент — **имя файла без расширения** (Excel + MD должны лежать в своих папках).  
Второй аргумент — **коэффициент скорости** (например `1.5`, `2`, `1.25`).

## Форматы MD-файлов

### Режим фильма (плоский формат)

Файл называется точно так же, как Excel: `Крупняк.md` → `Крупняк.xlsx`.  
Весь файл — один сплошной список таймкодов, заголовки `###` не нужны.

```
00:00:00:00 00:02:39:04 Заставка фильма. Титры.
00:01:19:03 00:02:08:12 Азартные игры. Казино.
00:01:38:16 00:01:52:11 Курение. Сигарета.
```

### Режим сериала (секционный формат)

Один MD-файл содержит несколько эпизодов, каждый — под заголовком `###`.  
Программа сама вычислит нужную секцию из имени Excel-файла и скорости.

```
### S2 E1 X1.5
00:00:00:00 00:00:02:16 Заставка студии.
00:00:26:05 00:00:28:13 Заставка серии.

### S2 E2 X1.5
00:00:00:00 00:00:02:16 Заставка студии.
00:00:02:17 00:00:04:19 Обнаженное тело. М.
```

Имя Excel → нужная секция:

| Excel файл | Скорость | Секция в MD |
|------------|----------|-------------|
| `FIRE_FORCE_S2_EP2.xlsx` | `1.5` | `S2 E2 X1.5` |
| `FIRE_FORCE_S3_EP12.xlsx` | `2` | `S3 E12 X2` |

## Как программа обрабатывает таймкоды

Таймкод имеет формат `ЧЧ:ММ:СС:КК` (кадры, FPS задаётся в конфиге, по умолчанию 25).

```
Позиция        = floor(начало × скорость)
Продолжительность = floor(конец × скорость) − floor(начало × скорость)
```

Пример при скорости 1.5 и FPS 25:

| Исходный таймкод | Кадры | × 1.5 | Результат |
|-----------------|-------|--------|-----------|
| `00:00:02:16` (начало) | 66 | 99 | `00:00:03:24` (позиция) |
| `00:00:04:19` (конец) | 119 | 178 | разница = 79 кадров = `00:00:03:04` (продолжительность) |

## Куда записываются данные в Excel

| Колонка | Содержимое |
|---------|-----------|
| A | № (порядковый номер) |
| B | Позиция (скорректированный старт) |
| C | Продолжительность (длительность сегмента) |
| D | Комментарий |

- Запись начинается со строки `default.start.row` (по умолчанию **13**).
- Колонки E и далее (справочные таблицы, заметки) **не трогаются**.
- Если `clear.existing.data=true` — старые данные в колонках A–D очищаются перед вставкой.

## Настройки (config.properties)

Файл `config.properties` рядом с JAR переопределяет встроенные настройки.

```properties
# Папка с .md файлами
md.files.dir=./md_files

# Папка с .xlsx шаблонами
excel.files.dir=./excel_files

# Имя листа в Excel-книге
default.sheet=Лист1

# Строка начала данных (1-индексация)
default.start.row=13

# Очищать ли колонки A–D перед записью
clear.existing.data=true

# Частота кадров для таймкодов
fps=25
```

## Обработка ошибок

| Ситуация | Сообщение |
|----------|-----------|
| Неверное число аргументов | `Ошибка: неверное количество аргументов` |
| Неверный формат скорости | `Ошибка: неверный формат скорости` |
| Excel-файл не найден | `Ошибка: Excel-файл "..." не найден` |
| MD-файл / секция не найдены | `Ошибка: Секция "..." не найдена` |
| Строка с некорректным таймкодом | Предупреждение в консоль, строка пропускается |

> Сообщение `ERROR StatusLogger Log4j2 could not find...` — это **не ошибка**, просто информационное сообщение внутренней библиотеки POI. На результат не влияет.
